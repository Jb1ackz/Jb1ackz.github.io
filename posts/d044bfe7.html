The Archetecture of ByteHouse

发表于2025-07-21|更新于2025-07-21
|浏览量:

Bycard文章中ByHouse的高层框架图

一、分层概览

ByteHouse 架构清晰分为三层，自顶向下依次是 Service Access Layer（服务接入层）、Compute Layer（计算层）、Storage Layer（存储层） ，各层职责明确，协同支撑数据查询与处理流程。

二、各层组件及功能解析

（一）Service Access Layer（服务接入层）

作为用户与系统交互的入口，负责接收查询、调度任务、管理元数据等，核心组件及作用如下：

  • Query Gateway：是查询请求的 “总闸口”，接收用户查询，协调后续流程，最终返回结果 ，是用户交互的核心入口。
  • Timestamp Oracle：提供时间戳服务，在需要同步操作（如保证数据一致性、事务处理时序）场景中，为操作标记精准时间戳，保障数据同步逻辑有序执行。
  • Daemon Manager：专注后台任务管理，像数据异步合并（Merge Out）、视图维护（View Maintenance ）这类非实时但需持续运行的任务，由它调度和监控，维持系统后台稳定运转。
  • Data Ingestor：承担批量数据导入职责，支持将外部数据批量写入 ByteHouse，为数据进入系统提供通道，适配离线数据批量加载场景。
  • Parser：对用户输入的查询语句（如 SQL）进行语法解析，将文本形式查询转化为系统可理解的结构化语法树，是查询处理的第一步 “翻译” 工作。
  • Catalog Manager：管理元数据，涵盖表结构（字段、类型、分区等）、数据库对象关系、数据存储位置映射等信息，为查询分析、优化提供元数据支撑。
  • Resource Manager：负责计算资源（如虚拟仓库的 CPU、内存等）的分配与监控，依据查询复杂度、数据量动态调配资源，确保资源高效利用，避免资源闲置或过载。
  • Query Analyzer and Optimizer：先分析查询语义、依赖，再基于成本模型（如基数估计、执行代价计算 ）优化查询，生成高效执行计划，决定查询怎么执行更 “快且省”。
  • Plan Scheduler：接收优化器产出的执行计划，调度到计算层的虚拟仓库执行，还会协调后台任务（如 Daemon Manager 管理的任务）的执行时机与资源，是任务执行的 “调度中枢”。

（二）Compute Layer（计算层）

聚焦查询计算，通过虚拟仓库（Virtual Warehouse）实现弹性计算，核心点：

  • Virtual Warehouse：是计算资源的 “容器”，可按需创建、销毁、扩缩容，每个虚拟仓库配 Cache，用于缓存热点数据（如频繁查询的结果、中间数据 ），加速查询，降低存储层访问压力，提升计算响应速度。多个虚拟仓库并行，支持多查询并发处理，实现计算资源隔离与弹性调度。

（三）Storage Layer（存储层）

负责数据持久化存储与管理，兼顾存储多样性与元数据维护：

  • Cloud Shared Storage：提供大规模、高可靠的共享存储，用于存放原始数据，支持数据的海量存储与多计算节点共享访问，适配云环境弹性扩展需求。
  • Columnar Storage：列式存储，按列组织数据，契合数据分析场景（如聚合、过滤操作 ），可大幅减少 IO、提升查询效率，是 OLAP 场景常用存储形式。
  • Metadata Key - Value Store：专门存储元数据（如 Catalog Manager 管理的表结构、分区信息等 ），以键值对形式快速读写，保障元数据访问高效性，支撑上层查询分析、优化流程。
  • Merge Out and View Maintenance：虽标注在计算层与存储层之间，实际是跨层的数据维护逻辑。Merge Out 负责将计算层缓存的增量数据、小文件合并成大文件，优化存储结构与查询性能；View Maintenance 维护视图（如物化视图 ），保证视图数据与底层数据一致，加速视图查询。

三、整体流程与优势

  • 流程：用户查询经 Query Gateway 进入，Parser 解析、Query Analyzer and Optimizer 优化，Plan Scheduler 调度到计算层虚拟仓库执行；数据从存储层读取，计算中用 Cache 加速，结果返回用户；后台任务（如数据导入、Merge Out）由 Daemon Manager 等组件协同管理。
  • 优势：分层解耦，各层专注自身职责，利于扩展与维护；计算层虚拟仓库弹性调度，适配不同负载；存储层混合存储（共享、列式、元数据键值存储 ），平衡存储成本与查询性能；元数据、缓存机制提升整体效率，支撑 ByteHouse 在大数据分析场景下高效运行 。
